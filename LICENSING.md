## FAQ

**Q: Can I self-host Lucid for my company?**
A: Yes, freely. AGPL only requires you to share source if you modify Lucid AND make it available over a network to users outside your organization.

**Q: Can I modify Lucid for internal use?**
A: Yes. Internal use within your organization does not trigger AGPL's source-sharing requirement.

**Q: Can I build a product that uses Lucid's API?**
A: Yes. Applications that communicate with Lucid over its API are separate works, not derivative works under AGPL.

**Q: Can I fork Lucid and sell it as a hosted service?**
A: Yes, but you must publish ALL of your source code (including modifications) under AGPL-3.0. If you want to keep modifications proprietary, you need a commercial license.

**Q: How is this different from Apache 2.0?**
A: Apache 2.0 allows anyone to take the code, modify it, and sell it as a closed-source service. AGPL requires that modifications to the code are shared with users when served over a network. This protects the open-source community while keeping the software genuinely open.

**Q: Why not BSL or source-available?**
A: We chose AGPL because it is a real open-source license (OSI-approved). Source-available licenses like BSL and ELv2 have led to community forks (OpenSearch, OpenTofu). We believe the open-source model, combined with strong managed services, is the right approach.
