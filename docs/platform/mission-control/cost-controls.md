# Cost Controls

Cost controls let you set spending limits on your agents to prevent runaway costs. Limits are enforced automatically — when a limit is hit, the agent pauses until the limit resets.

## Available Limits

| Limit | Scope | Resets |
|-------|-------|--------|
| **Per-run limit** | Maximum cost for a single agent run | Per conversation turn |
| **Daily limit** | Maximum cost per calendar day (UTC) | Midnight UTC |
| **Monthly limit** | Maximum cost per calendar month | First of the month |

## Setting Cost Limits

1. Go to your agent's detail page
2. Click the **Guardrails** tab
3. Set dollar amounts for per-run, daily, and/or monthly limits
4. Save

All limits are in USD. You can set any combination — for example, a daily limit without a per-run limit, or all three.

## How Enforcement Works

Cost tracking happens in real-time during agent execution:

1. Before each tool call, the system checks accumulated cost against limits
2. If a limit would be exceeded, the tool call is blocked
3. The agent is informed it has reached its cost limit
4. The agent pauses until the relevant limit resets

### What Counts as Cost

- **Input tokens** — Tokens sent to the AI model
- **Output tokens** — Tokens generated by the AI model
- **Embedding tokens** — Tokens used for memory and knowledge base embeddings

Tool execution (API calls, web searches) does not directly count toward cost limits — only AI model token usage is tracked.

## Viewing Cost Data

### Agent-Level Costs

On the agent's detail page, the Guardrails tab shows:
- Current spend vs limits (per-run, daily, monthly)
- Historical spending trend

### Fleet-Level Spend

The **Spend** page in Mission Control shows:
- Total fleet spending by day/week/month
- Cost breakdown by agent
- Cost per conversation
- Optimization recommendations

## Cost Optimization

Mission Control's cost optimizer analyzes your fleet and suggests:

- **Model downgrades** — Agents using expensive models for simple tasks
- **Memory strategy adjustments** — Reducing extraction frequency for low-value conversations
- **Plugin consolidation** — Removing unused plugins that add overhead

## Loop Detection

Agents stuck in loops (calling the same tool with the same arguments repeatedly) are automatically detected and paused. This prevents a single stuck conversation from burning through your budget.

The loop detector triggers when the same tool+arguments combination appears more than 3 times in a single run.
